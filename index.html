Projects

1. Web Management Letta School — Fullstack School Management System

Stack: React · Go Fiber · PostgreSQL · Docker · AWS EC2 · GitHub Actions · GitLeaks · Semgrep · Trivy · OWASP ZAP · CloudTrail  |  Live Demo →  |  GitHub →

Letta School adalah sistem manajemen sekolah berbasis web yang saya bangun dari nol sebagai proyek fullstack komprehensif. Sistem ini menangani kebutuhan operasional sekolah meliputi manajemen data siswa, guru, kelas, mata pelajaran, absensi, dan laporan akademik dalam satu platform terintegrasi dengan antarmuka yang bersih dan responsif.

Kenapa Go Fiber untuk backend? Saya memilih Go Fiber karena performanya yang jauh superior dibandingkan framework Node.js. Benchmark menunjukkan Go Fiber mampu menangani hingga 5 juta request per detik pada hardware standar, dengan latency rata-rata di bawah 1ms. Ini membuat backend tetap responsif bahkan saat banyak pengguna mengakses sistem secara bersamaan. GORM digunakan sebagai ORM untuk menyederhanakan query PostgreSQL tanpa mengorbankan fleksibilitas.

Sistem autentikasi dan otorisasi: Implementasi JWT (JSON Web Token) untuk session management tanpa state di server. Role-based access control (RBAC) membedakan hak akses tiga jenis pengguna: admin (akses penuh), guru (manajemen kelas dan nilai), dan siswa (view-only untuk data pribadi dan jadwal). Setiap API endpoint dilindungi middleware autentikasi yang memvalidasi token dan memeriksa role sebelum memberikan akses.

DevSecOps Pipeline yang komprehensif: Ini adalah aspek yang paling membedakan proyek ini dari portofolio mahasiswa kebanyakan. Setiap push ke branch main secara otomatis menjalankan pipeline GitHub Actions dengan 6 tahap keamanan:

  1. GitLeaks — mendeteksi API key, password, atau secret yang tidak sengaja ter-commit ke repository
  2. Semgrep — Static Application Security Testing untuk mendeteksi kerentanan keamanan di source code Go dan JavaScript
  3. Dependency Audit — npm audit untuk frontend dan go mod tidy untuk backend, memastikan tidak ada dependency dengan CVE yang diketahui
  4. Trivy — scanning container image Docker sebelum push ke registry, mendeteksi vulnerability di OS packages dan library
  5. OWASP ZAP — Dynamic Application Security Testing setelah deployment ke staging, mensimulasikan serangan nyata seperti SQL injection, XSS, dan CSRF
  6. AWS CloudTrail — audit log semua aktivitas di infrastruktur AWS untuk forensik dan compliance

Aplikasi ini live dan dapat diakses di school.lananuranf.my.id.

2. Sales Data Dashboard — Data Science & ML Analytics

Stack: Python · Streamlit · Pandas · Matplotlib · Scikit-learn · Random Forest  |  Live App →  |  GitHub →

Proyek data science end-to-end yang menganalisis performa penjualan historis dan membuat prediksi untuk periode berikutnya menggunakan algoritma machine learning. Dashboard ini dirancang untuk membantu bisnis memahami tren penjualan, mengidentifikasi produk dan kategori terlaris, serta mengambil keputusan berbasis data.

Pipeline data: Raw data diproses melalui tahap cleaning menggunakan Pandas — handling missing values dengan strategi yang sesuai konteks (mean imputation untuk numerik, mode untuk kategorik), deteksi dan penanganan outlier dengan IQR method, normalisasi format tanggal, dan feature engineering untuk mengekstrak variabel seperti day of week, month, quarter, dan lag features untuk time-series analysis.

Model machine learning: Random Forest dipilih karena kemampuannya menangani data tabular dengan banyak fitur tanpa memerlukan feature scaling, robust terhadap outlier, dan memberikan feature importance yang dapat diinterpretasi bisnis. Model dilatih dengan cross-validation 5-fold untuk memastikan generalisasi yang baik. Hyperparameter tuning dilakukan dengan RandomizedSearchCV untuk mencapai akurasi optimal.

Dashboard live di: sales-dashboard-lananuranf.streamlit.app

3. Dockerized E-Commerce App — Cloud-Native Fullstack Platform

Stack: React · Go Gin · GORM · PostgreSQL · Docker Compose · AWS EC2 · Nginx  |  GitHub →

Platform e-commerce production-grade yang dibangun dengan prinsip cloud-native dan fully containerized menggunakan Docker. Proyek ini fokus pada arsitektur yang repeatable, scalable, dan mudah dimaintain — dengan setiap service berjalan dalam container terpisah yang diorkestrasikan dengan Docker Compose.

Arsitektur layered backend: Go Gin dengan GORM diorganisir menggunakan clean architecture: handler layer menerima HTTP request dan mengembalikan response, service layer berisi business logic murni tanpa ketergantungan infrastruktur, dan repository layer mengabstraksikan semua operasi database. Pemisahan ini membuat setiap layer bisa di-unit test secara independen dan memudahkan penggantian komponen tanpa mengubah business logic.

Infrastructure as configuration: Seluruh infrastruktur didefinisikan dalam docker-compose.yml — React Nginx container untuk frontend, Go backend container, dan PostgreSQL container dengan persistent volume. Environment variables di-inject via .env file yang tidak pernah di-commit ke repository. Nginx berfungsi sebagai reverse proxy yang menangani SSL termination, static file serving, dan routing ke backend.

4. AI Agent — Telegram Personal Assistant

Stack: Python · Groq LLaMA 3.3 · Telegram Bot API · Whisper V3 · LLaMA Vision · scikit-learn · python-pptx · BeautifulSoup4 · Google Calendar API · Railway

Personal AI assistant yang berjalan 24/7 di Telegram, menggabungkan berbagai kapabilitas AI generatif dan otomasi dalam satu platform. Proyek ini adalah eksplorasi saya dalam membangun agentic AI system yang dapat memahami context, menggunakan berbagai tool, dan menyelesaikan task kompleks secara otonom.

Kapabilitas AI dan multimodal: Voice message transcription menggunakan Whisper V3 dengan akurasi tinggi untuk Bahasa Indonesia dan Inggris. Image analysis menggunakan LLaMA Vision untuk mendeskripsikan, menganalisis, atau mengekstrak informasi dari gambar yang dikirim ke bot. LLM backbone menggunakan Groq LLaMA 3.3 melalui Groq API yang memberikan inferensi sangat cepat (token per detik yang jauh lebih tinggi dari OpenAI API) dengan biaya yang lebih rendah.

Fitur productivity dan otomasi: Auto-generate presentasi PowerPoint dari outline teks menggunakan python-pptx; analisis dataset machine learning langsung dari file CSV yang diupload (Random Forest + Logistic Regression dengan report otomatis); web scraping dengan BeautifulSoup4; integrasi dua arah Google Calendar untuk manajemen jadwal; habit tracking dengan pattern recognition; pencarian jurnal ilmiah di ArXiv; Python code executor untuk menjalankan script kecil; YouTube downloader; dan Pomodoro timer.

Bot ini di-deploy di Railway dengan auto-deploy setiap push ke branch main, memastikan update fitur langsung tersedia tanpa downtime atau manual deployment.
